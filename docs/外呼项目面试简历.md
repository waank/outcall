# 项目介绍
智能外呼平台 是一款企业级数智化运营系统，通过 AI Agent 技术实现与客户实时建立语音通话的智能调度平台。系统广泛应用于金融催收、账单通知、营销推广等核心业务场景，帮助企业构建多场景、全流程的 AI 数字员工体系，显著提升运营效率和客户触达率。
外呼调度模块是发起呼出的控制中枢，通过定时任务+异步线程池+状态机等策略，构建高并发、可伸缩、可观测的外呼调度与执行。系统日均处理数百万级外呼任务，支持跨天时间窗口、优先级管理、限流背压等复杂业务场景。
# 核心能力
1. 并行调度：将整个呼叫流程分为5个阶段：收单-〉分组-〉分发-〉执行-〉重试，每个阶段由专属且隔离的线程池定时驱动，形成异步化、非阻塞的调度流水线，通过号码队列及队列组数据的状态流转实现任务处理的全链路流水线化并行。countDownLatch (map-reduce) 
2. 优先调度：提出用户画像与调度系统深度融合的择时调度机制，通过预测用户接听概率，精准定位最佳拨打时段，在分组阶段构建"择时优先组"，由定时任务触发准时分发实现即时响应 + 插队调度机制，确保实时性提升呼叫接通率上升3%。
3. 弹性调度：通过多级限流机制（全局限流、租户限流、任务限流）、线程池动态扩容、队列阈值、超时重置等方式适配大流量低频任务（千万级/短窗口）与小流量高频任务，防止单一任务阻塞或资源占用，实现资源利用率与调度效率的平衡。
4. 呼叫执行：通过定时任务或者分发阶段的触发，通过redis的队列分发机制，每台机器自主地从缓存中获取队列组，将该组下的号码加载到线程池执行呼出，每个任务可以在多台机器同时执行；使用 CountDownLatch实现组内队列并发执行与等待，将串行处理改为并行处理，通过水平扩展实现呼叫性能的线性增长。 kill  pid 优雅关闭 （jvm registerHook(()->{.shutdown()}) task -> wal -> status
5. 平滑限流：集群环境下多节点同时执行外呼任务，同一时刻流量集中爆发，导致中间件压力骤增、下游系统过载、限流呼叫失败等问题。需要设计平滑限流机制，分散流量峰值，保障系统稳定运行。
   （1）实现Redis计数器限流算法：以秒级时间戳构建缓存Key，实现精确的时间窗口限流；使用Redis原子操作INCR保证线程安全的计数；设置2秒自动过期，避免内存泄漏
   （2）触发时间扰乱：定时器启动时加入30秒内随机睡眠；打散多节点启动时间，避免同时刻流量峰值
   （3） 多级匀速供给：获取队列组前和执行呼叫前：对QPS限流检查，每级检查均带超时机制，保障数据供给速度适中；放入线程池前执行队列长度阈值检查。
   (4) 分组大小调优。分组过大：DB压力大、线程池过载；分组过小：供给不足、并发效率低；通过多轮压测，确定最优分组大小为300条。保障流量峰值平滑分散，系统在高峰期稳定运行，无OOM或雪崩。

6. 数据一致性：由于每个阶段都是异步并行处理，数据状态可能发生错乱，进一步导致重呼/漏呼，因此线程执行时需严格校验当前数据状态是否符合预期。采用以下三种策略，累计保障亿级数据的正确性。
   （1）重试机制：异常队列自动生成重试组、支持最大重试次数控制、重试组类型标记为RETRY。
   （2）补偿机制：定时扫描处理中队列、根据通话记录回写最终状态、处理未正确更新的悬挂记录。
   （3）流程机制：通过一查二锁三校验的方式实施，不符合预期的数据进行监控预警，人工兜底排查处理。
